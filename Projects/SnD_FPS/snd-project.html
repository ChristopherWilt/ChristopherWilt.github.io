PROJECT OVERVIEW

This project is a modular First-Person Shooter framework engineered in C++ to serve as a scalable foundation for competitive multiplayer games.

Unlike standard template projects, this framework integrates Epic Online Services (EOS) for peer-to-peer matchmaking, lobby management, and secure user authentication. The goal was to build a reusable backend that supports Triple-A quality mechanics with low-latency network performance.

PROJECT PROGRESS

The current status of the project includes a fully functional FPS Player Controller (First Person and Third person meshes), weapon systems, health and damage mechanics, weapon pickups & dropping, BuyMenu (For SND Games), as well as Animation Systems to drive the character's movement and actions on both meshes. All with a focus on modularity and reusability with server-authoritative in mind.

On the networking side, i am currently implementing EOS systems with EIK. Which includes Lobby Creation/Joining, Matchmaking, and Player Authentication. So far, the lobby and matchmaking systems are functional, allowing players to create and join sessions seamlessly. The next steps involve integrating these systems with the gameplay mechanics to ensure smooth multiplayer experiences. (Note: The Online Subsytems are still being Developed and Debugged, so the files are not on the source just yet.)

NETWORK REPLICATION

The core challenge of this project was implementing server-authoritative movement and state synchronization.

I utilized Unreal Engine's replication graph to manage bandwidth efficiently. The weapon system features client-side prediction for instant feedback, with server reconciliation to prevent desynchronization. Animation states are replicated via a compressed integer flag system to minimize packet size while maintaining visual fidelity across all clients.

MODULAR WEAPON SYSTEM

I architected a data-driven AWeapon class designed for extensibility. Instead of hardcoding behavior, weapons utilize a Fire Configuration Struct that dictates Fire Modes (Single, Burst, Auto), Recoil Patterns, and Damage Falloff.

The system uses a slot-based Inventory Component that handles equipment logic server-side. This ensures that a player cannot equip a weapon they haven't purchased or picked up, preventing cheat exploits while maintaining a flexible "Loadout" system.

CLIENT-SIDE PREDICTION

In a competitive shooter, "feeling" responsive is paramount. I implemented Client-Side Prediction for the firing mechanics.

When a player clicks fire, the client immediately plays the muzzle flash and calculates the hit trace locally. The Server_FireShot RPC then validates ammo and recalculates the hit using Lag Compensation logic. The server creates the authoritative state, but the client never waits for round-trip latency to see feedback.

DYNAMIC UI & ECONOMY

I engineered a responsive Buy Menu System using C++ and UMG. The system dynamically populates weapon columns based on a WeaponsForSale array, allowing designers to add new guns without touching code.

The UI logic includes real-time Affordability Checks. In `NativeConstruct`, the widgets bind to player delegates to track money. If a player cannot afford an item, the button is automatically disabled and greyed out, providing instant feedback and preventing invalid server requests.

MOVEMENT STATE MACHINE

To achieve "Triple-A" fluidity, I extended the standard Character Movement Component to support advanced states like Sliding.

The sliding mechanic dynamically alters ground friction and braking deceleration in real-time. I implemented a custom `ApplySlidePhysics()` function that shrinks the capsule hitbox and applies a velocity impulse. This state is replicated via Server_StartSliding, ensuring the hitbox matches the visual mesh on all clients.
