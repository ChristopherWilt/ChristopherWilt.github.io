PROJECT OVERVIEW

This project is a modular First-Person Shooter framework engineered in C++ to serve as a scalable foundation for competitive multiplayer games.

Unlike standard template projects, this framework integrates Epic Online Services (EOS) with EIK for peer-to-peer matchmaking, lobby management, and secure user authentication. So far, the lobby and matchmaking systems are functional, allowing players to create and join sessions seamlessly. (Note: The Online Subsystems are still being developed and debugged, so the files are not on the source just yet.)

MODULAR HUD & ECONOMY

I engineered a responsive Buy Menu System using C++ and UMG, which dynamically populates weapon columns based on a WeaponsForSale array. The UI logic includes real-time affordability checks; if a player cannot afford an item, the button is automatically disabled, preventing invalid server requests.

Beyond the economy, the UI utilizes a Master HUD Architecture (UModularPlayerHUD) to control gameplay widgets. This includes a procedural crosshair that dynamically scales based on movement and fire-spread, and a networked minimap that uses a Multicast_PingMinimap RPC to broadcast enemy locations when unsuppressed weapons are fired. The tactical Killfeed was optimized using TWeakObjectPtr and timer handles to gracefully manage message decay and prevent memory leaks.

Again everything in the system is designed to be modular and data-driven, allowing for easy adjustments to the scaling behavior and visual style without modifying core logic. The Minimap, for example, is built to be flexible; the same ping system can be repurposed for tactical equipment like flashbangs or heartbeat sensors by simply changing the icon and parameters in the RPC call, demonstrating the framework's adaptability for future feature expansion. The Mini-Map also shows FOV indicators that dynamically adjust based on the player's current weapon, providing critical situational awareness in combat scenarios or when a you or a team-mate is aiming down sights.

DYNAMIC COMPASS SYSTEM

To provide precise spatial awareness, I engineered a dynamic top-bar compass system. The C++ backend feeds the player's normalized and absolute yaw to the UI, but the UI layer handles the heavy lifting of visual formatting using optimized Blueprint math.

Instead of expensive tick-based conditional branches, the center text calculator uses a highly optimized modulo algorithm. By adding an offset of 22 to the player's yaw, dividing by 45, and applying modulo 8, the 360-degree rotation is instantly converted into an index (0-7) that drives a Select node for the 8 cardinal directions.

The sliding strip markers are generated dynamically. The widget calculates if a specific tick falls on a 45-degree interval using modulo division; if it does, a Widget Switcher swaps the standard degree number for its corresponding cardinal letter, resulting in a clean, modern tactical display.

ADVANCED WEAPON SYSTEM

I architected a data-driven AWeapon class designed for extensibility, utilizing a Fire Configuration Struct for modes and recoil. The standout feature is the "True Bore Alignment" ADS system. This automated calibration function freezes component updates and utilizes Quaternion math (FQuat::FindBetweenNormals) to perfectly align any weapon’s barrel vector with the camera’s forward vector at runtime, procedurally zeroing any optic.

The inventory relies on a UInventoryComponent that supports a universal weapon slot toggle, enabling modular loadout swapping. Physics-based interactions allow dropped weapons to use randomized "Pop and Spin" torque. Players interact with these items via an optimized proximity trace that calculates a dot-product "look-score". Furthermore, weapon impacts are optimized using a Global Decal Pool capped at 50 instances to strictly preserve rendering budgets.

MOVEMENT & PREDICTION

To achieve "Triple-A" fluidity, I extended the Character Movement Component to support states like Sliding. This dynamically alters ground friction and shrinks the capsule hitbox via a custom ApplySlidePhysics() function. The state is replicated securely, while procedural arm interpolation (FMath::VInterpTo) blends the raw input sway into the first-person mesh to create a responsive combat "glide."

In a competitive shooter, feeling responsive is paramount. I implemented Client-Side Prediction for the firing mechanics. When a player fires, the client immediately calculates the hit trace locally for zero-latency feedback. The Server_FireShot RPC then validates ammo and recalculates the hit using Lag Compensation logic. Animation states are replicated via a compressed integer flag system to minimize packet size while maintaining exact visual fidelity.

DEVELOPMENT ROADMAP

The framework is continuously evolving. The next phase of development focuses on expanding the combat sandbox, refining player locomotion, and scaling the architecture to support multiple game modes.

  • Advanced Weapon Mechanics: Implementing procedural weapon sway, wall-collision avoidance, and a robust attachment system, alongside tactical and lethal equipment integration.
  • Locomotion & Player State: Adding a mantle system, networked spectator states, full-body animation refinements, and configurable health regeneration.
  • Expanded Game Modes: Engineering a dedicated C++ Bomb class to finalize Search and Destroy, while scaling the backend to support Team Deathmatch, Free-For-All, and Capture the Flag.
  • UI & Social Integration: Completing the out-of-match experience with a Start Menu, Lobby System, Friends List, and advanced HUD elements (multi-kill tracking, stance indicators, and stacked damage numbers).
